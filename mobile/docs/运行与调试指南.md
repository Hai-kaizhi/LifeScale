### 首次构建一次性准备（中国大陆网络）

首次 `flutter build` 会下载 Gradle 发行包与 AGP/AndroidX 依赖。为避免在墙内网络长时间卡住，请按需执行：

1. **让 Flutter 认到 Android SDK**（一次性）：
   ```powershell
   flutter config --android-sdk E:/AndroidSdk
   flutter doctor --android-licenses   # 全部 y，接受许可
   ```
2. **Maven 依赖镜像（已预置）**：`android/settings.gradle.kts` 已把阿里云镜像置于 `google()/mavenCentral()` 之前，AGP/Kotlin/AndroidX 依赖会走镜像（快）。
3. **Gradle 发行包镜像（按需）**：Gradle 发行包本身从 `services.gradle.org` 下载，不被 Maven 镜像覆盖。若首次卡在下载 `gradle-8.12-all.zip`，把 `android/gradle/wrapper/gradle-wrapper.properties` 的 `distributionUrl` 临时改为：
   
   ```properties
   distributionUrl=https\://mirrors.cloud.tencent.com/gradle/gradle-8.12-all.zip
   ```
   下载完成（`~/.gradle/wrapper/dists/.../gradle-8.12-all.zip` 生成）后可恢复原值。
4. **编译目标**：`android/app/build.gradle.kts` 已锁定 `compileSdk/targetSdk = 35`（匹配本机已装的 `platforms/android-35`，避免触发 android-36 平台下载）。应用仍可在 Vivo X100（API 36）运行。

## 二、先启动后端


`start.bat` 会起 PostgreSQL/Redis + 后端 + 桌面端。移动端只依赖后端（8080）。

## 三、API base 与明文（cleartext）

后端是 `http://`（非 HTTPS）。Android 9+ 默认拦截明文，故：

- **模拟器** 用 `http://10.0.2.2:8080/api`（`10.0.2.2` 映射到宿主机回环）。
- **真机** 用开发机 LAN IP，如 `http://192.168.x.x:8080/api`（手机与电脑同一局域网）。

已通过 `android/app/src/main/res/xml/network_security_config.xml` 对 `10.0.2.2`、`localhost`、`192.168/10./172.` 段放通明文。生产环境应切 HTTPS 并收紧该配置。

API base 通过 `--dart-define=LIFESCALE_API_BASE_URL=...` 编译期注入（见 `lib/core/config/app_config.dart`）。一键脚本会自动探测 LAN IP，无需手改。

## 四、一键脚本（推荐）

在 **cmd 或 PowerShell** 中执行（不是 Git Bash）：

```powershell
cd E:\AINewProject\LifeScale\code\mobile\scripts
.\run.bat           # 开发调试：预检后端 → pub get → build_runner → 选目标 → flutter run
.\boot_emulator.bat # 仅启动 API 36 模拟器
.\build_apk.bat     # 出 release APK（真机侧载）
```

`run.bat` 行为：
1. 预检后端 `/api/health`（未起则提示先跑 `start.bat`）。
2. `flutter pub get`（失败重试）+ `dart run build_runner build`。
3. `adb devices` 选目标——**检测到真机**则注入 LAN IP；**否则**启动模拟器并用 `10.0.2.2`。
4. `flutter run`（前台，保留 `r` 热重载 / `q` 退出）。
5. 过程日志：`code/logs/mobile.log`。

> 脚本中 `E:\AndroidSdk`、`E:\java21`、AVD 名为本机硬编码；换机时改脚本顶部变量即可。

## 七、真机 Vivo X100（Android 16）流程

### 7.1 一次性：开启开发者与调试

1. 设置 → 关于手机 → 软件版本信息 → 连续点「**编译编号 / 版本号**」7 次，开启开发者选项。
2. 设置 → 系统 → 开发者选项，开启：
   - **USB 调试**
   - **USB 调试（安全设置）**（Vivo 专有，安装/调试需要）
   - **仅充电模式下允许 ADB 调试**（Vivo 专有）
   - **USB 安装 / 允许通过 USB 安装应用**（**Vivo 默认禁用未签名 APK 安装，必须开**，否则侧载被拦）
3. 用数据线连电脑（选「传输文件 / MTP」模式）。
4. 手机弹出「允许 USB 调试？」→ 确定。如已拒绝：开发者选项里「撤销 USB 调试授权」后重连。

### 7.2 驱动

- 本机已装 **Vivo PC Suite**（`D:\VivoWork\pcsuite`），通常自带驱动。
- 若 `adb devices` 显示 `unauthorized` 或不识别，用 Vivo PC Suite 重装驱动；或换数据线/USB 口。

### 7.3 调试运行（带热重载）


### 7.4 侧载 release APK（独立运行）

## 九、常见排错

| 现象 | 原因 / 处理 |
| --- | --- |
| 脚本报「后端未就绪」 | 没起后端。先 `code\start.bat`。 |
| 登录/接口失败、状态行 `api:` 不对 | API base 没对：模拟器必须是 `10.0.2.2`，真机必须是开发机 LAN IP。看状态行核对。 |
| 真机连不上后端（超时） | 手机与电脑不同局域网 / 防火墙拦 8080 / cleartext 域段不含你的 IP 段（改 `network_security_config.xml`）。 |
| cleartext 被拦（`CLEARTEXT communication ... not permitted`） | 确认 `AndroidManifest` 指向 `network_security_config`，且配置含对应域。 |
| `adb devices` 显示 `unauthorized` | 手机端授权弹窗未确认；或「撤销 USB 调试授权」后重连。 |
| Vivo 装 APK 失败 / 一闪而过 | 没开「USB 安装 / 允许通过 USB 安装应用」。 |
| Gradle 构建慢 / 失败 | 首次会下载 Gradle 8.12 发行包（约 180MB）+ AGP/AndroidX 依赖。**依赖**已走 `android/settings.gradle.kts` 配置的阿里云镜像（快）；**Gradle 发行包**走 `services.gradle.org`，中国大陆可能慢。若卡住，把 `android/gradle/wrapper/gradle-wrapper.properties` 里的 `distributionUrl` 改为腾讯镜像：`https\://mirrors.cloud.tencent.com/gradle/gradle-8.12-all.zip`，下载完成后恢复原值即可。 |
| Java/Gradle 冲突 | 确认 `JAVA_HOME=E:\java21`；脚本已设。必要时锁 `compileOptions` 为 17（已锁）。 |
| `build_runner` 拉包慢 | 中国镜像已配；可 `flutter pub get --offline`（已缓存时）。 |
| 改了 freezed 模型编译报错 | 跑 `dart run build_runner build --delete-conflicting-outputs`。 |

## 十、工程结构速览（便于后续阶段扩展）

```
lib/
  main.dart                  入口：ProviderScope + bootstrap
  app/                       app.dart / router.dart / bootstrap.dart
  core/
    config/app_config.dart   --dart-define 读 API base
    network/                 api_client（dio + 拦截器）、api_result、dto/、auth_bus
    storage/                 app_paths、secure_token_store、prefs_store、database_service、vault_storage
    util/                    crypto_util、id_util、date_util
    providers.dart           Riverpod 核心提供者
  features/
    auth/                    data/api+repository、domain/auth_user、presentation/auth_controller+state
    vault/                   data/api+repository、presentation/foundation_page+controller+state（第 0 步验收页）
    daily_markdown/          domain（模型）、data（parser/serializer/factory）
  shared/constants/markdown.dart  文法常量（段标题/占位/类别）
```

后续阶段（登录/今日/记录/复盘/日历/笔记/附件/离线队列/闭环）在 `features/` 下新增模块，复用 `core/network` 与 `core/storage`，`sync_state` 表与完整同步引擎在阶段一接入。
