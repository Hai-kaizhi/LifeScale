# 安全策略

## 报告漏洞

如果你发现安全漏洞，请**不要**开公开的 Issue。

请通过以下方式私密报告：

- 在 Gitee 或 GitHub 仓库创建一个**保密（Private）Issue**（如平台支持），或
- 直接联系仓库维护者

报告时请包含：

- 漏洞的详细描述
- 复现步骤（最小可复现示例最佳）
- 影响范围评估
- 你期望的披露时间（如需协调发布）

## 响应承诺

- 收到报告后 **48 小时内**确认收到
- 尽快评估漏洞严重程度并制定修复方案
- 在修复版本发布前不公开漏洞细节
- 修复发布后，会在 CHANGELOG.md 中记录（不含漏洞利用细节）

## 支持的版本

仅对最新发布版本（最新 tag）提供安全更新。旧版本的安全修复不保证回移植。

## 数据安全说明

LifeScale 是**纯本地应用**：所有数据（日程、笔记、复盘、附件）保存在你的设备本地，不联网、不上传任何服务器。因此：

- **数据归属**：数据完全属于你，存在本地 SQLite 与本地 Markdown 文件中。
- **卸载即清除**：卸载应用会删除其本地数据，请定期手动备份本地数据目录。
- **无远程攻击面**：应用不监听任何网络端口、不发起任何网络请求，不存在服务端可被远程利用的漏洞。
- **不要提交个人数据**：仓库内不含用户数据；如果你 fork 后运行应用产生的本地数据，请勿提交到 git（用户数据目录不在仓库内）。
