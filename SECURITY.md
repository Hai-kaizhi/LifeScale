# 安全策略

## 报告漏洞

如果你发现安全漏洞，请**不要**开公开的 Issue。

请通过以下方式私密报告：

- 在 Gitee 或 GitHub 仓库创建一个**保密（Private）Issue**（如平台支持），或
- 直接联系仓库维护者

报告时请包含：

- 漏洞的详细描述
- 复现步骤（最小可复现示例最佳）
- 影响范围评估
- 你期望的披露时间（如需协调发布）

## 响应承诺

- 收到报告后 **48 小时内**确认收到
- 尽快评估漏洞严重程度并制定修复方案
- 在修复版本发布前不公开漏洞细节
- 修复发布后，会在 CHANGELOG.md 中记录（不含漏洞利用细节）

## 支持的版本

仅对最新发布版本（最新 tag）提供安全更新。旧版本的安全修复不保证回移植。

## 安全部署提醒

自托管部署时务必注意：

- `.env` 中所有 `ChangeMe` 占位符必须替换为强随机值（后端会对弱 JWT secret / 弱 bootstrap 密码 fail-fast 拒绝启动）
- 生产环境启用邀请码注册和限流（`.env.production.example` 默认开启）
- 不要将 `.env`、SSL 证书、`backend/data/`（含用户笔记正文）提交到任何版本控制或公开位置
