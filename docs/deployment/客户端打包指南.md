# 客户端打包指南

> 本文说明如何打包桌面端（Windows 安装包）和移动端（Android APK）。
> 产物输出到仓库外的 `workspace/release/`，不入 git。

## 桌面端打包（Tauri → Windows 安装包）

### 手动打包

```bash
cd desktop
pnpm install
pnpm tauri build          # 产出 src-tauri/target/release/bundle/
```

### 脚本打包（推荐，Windows）

```bash
cd scripts/dev
./build-desktop.bat -Version 0.2.0 -Channel beta        # 测试版（NSIS exe）
./build-desktop.bat -Version 0.2.0 -Channel release      # 正式版（NSIS exe）
./build-desktop.bat -Version 0.2.0 -Channel release -IncludeMsi   # 正式版连 msi 一起打
```

脚本会自动：注入版本号到 `tauri.conf.json` → 构建 → 归档产物到 `workspace/release/desktop/<渠道>/` → 生成 BUILD-INFO.txt。

> 产物位置：`src-tauri/target/release/bundle/`（构建产物，gitignore）。脚本会把最终安装包复制到 `workspace/release/desktop/`。

## 移动端打包（Flutter → Android APK）

### 手动打包

```bash
cd mobile
flutter pub get
flutter build apk --flavor beta --release     # 测试版（beta flavor）
flutter build apk --flavor prod --release     # 正式版（prod flavor）
```

产物位置：
- 测试版：`mobile/build/app/outputs/flutter-apk/app-beta-release.apk`
- 正式版：`mobile/build/app/outputs/flutter-apk/app-prod-release.apk`

（构建产物，gitignore）

### 脚本打包（推荐，Windows）

```bash
cd scripts/dev
./build-android.bat -Version 0.2.0 -Channel beta        # 测试版（beta flavor）
./build-android.bat -Version 0.2.0 -Channel release      # 正式版（prod flavor）
./build-android.bat -Version 0.2.0 -Channel beta -BuildNumber 5   # 指定构建号
./build-android.bat -Version 0.2.0 -Channel beta -InstallToPhone   # 打包并安装到已连接真机
```

脚本会自动：注入版本号到 `pubspec.yaml` → `flutter build apk --flavor <beta|prod> --release` → 归档到 `workspace/release/android/<渠道>/` → 生成 BUILD-INFO.txt。

> **Flavor 架构**：`mobile/android/app/build.gradle.kts` 配置了 `productFlavors`：
> - **prod**：正式版，默认 `applicationId`，无后缀
> - **beta**：测试版，`applicationId` 追加 `.beta`（可与正式版**同机共存**），版本名加 `-beta`，应用名带「测试版」
>
> 两套图标/应用名物理隔离。渠道（beta/release）→ Gradle flavor 映射：beta→beta、release→prod。

## 应用图标

### 桌面端

```bash
cd scripts/dev
./gen-icons.bat           # 根据 desktop/src-tauri/app-icon.png 重新生成全套图标
```

源图：`desktop/src-tauri/app-icon.png`。脚本用 `pnpm tauri icon` 生成 `src-tauri/icons/` 下全套尺寸（icon.ico/icns/png/Square*）。

### 移动端

```bash
cd scripts/dev
./gen-icons.bat                 # 生成移动端 + 桌面端全套
./gen-icons.bat -AndroidOnly    # 仅移动端（prod + beta 两套）
```

源图（修改后跑脚本刷新所有尺寸）：
- 正式版：`mobile/assets/icons/brand/app_icon_source_1024.png`
- 测试版：`mobile/assets/icons/brand/app_icon_source_1024_beta.png`

脚本用 `flutter_launcher_icons`（配置：`mobile/flutter_launcher_icons-prod.yaml` / `-beta.yaml`）生成 `android/app/src/{main,prod,beta}/res/mipmap-*` 三套图标。

> 项目尚未引入 `flutter_launcher_icons`（需在 `pubspec.yaml` 加依赖 + 配置 `flutter_launcher_icons.yaml`）。待 flavor 体系落地后再补脚本化生成。

## 签名

- **Android 签名**：当前 release 包使用 **debug 签名**（仅限内测/自用安装）。上架应用商店前必须换成正式 keystore：
  - keystore 文件放在 `workspace/secrets/`（绝不入仓）
  - 在 `mobile/android/key.properties`（gitignore）配置签名信息
  - 在 `build.gradle.kts` 的 `release` buildType 引用 signingConfig
- **Windows 签名**：当前安装包未做代码签名，首次安装会被 SmartScreen 拦截（需手动「更多信息 → 仍要运行」）。正式分发前建议接入代码签名证书，证书放 `workspace/secrets/`。

真实签名材料永远只在 `workspace/secrets/`，仓库内不含任何密钥/证书。

## 版本号

版本号在各端配置中维护，发版时两端保持一致：
- 桌面端：`desktop/src-tauri/tauri.conf.json`（version）+ `desktop/package.json`
- 移动端：`mobile/pubspec.yaml`（version: x.y.z+buildNumber）

并在 `CHANGELOG.md` 记录每个版本的变更。
